# Security Policy

## Supported Versions

Security updates are currently provided for the latest `main` branch and latest tagged release.

| Version | Supported |
| --- | --- |
| Latest | Yes |
| Older versions | No |

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use one of these private channels:

1. GitHub Security Advisories (preferred, private by default)
2. Private maintainer contact if provided by the repository owner

Include:

- Clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Suggested fix (if available)

## Response Targets

- Initial acknowledgment: within 72 hours
- Status update: within 7 days
- Fix timeline: depends on severity and complexity
